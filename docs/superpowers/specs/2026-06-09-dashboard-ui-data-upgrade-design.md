# 世界杯竞彩分析系统 UI 与数据升级设计

日期：2026-06-09

## 背景

当前系统已经可以导入 500.com 赔率 CSV，运行 Poisson/xG/Elo/市场边际模型，并在 Streamlit 页面展示每日筛选和单场分析。用户提出四项升级诉求：

1. 网页 UI 需要美化。
2. 国家队需要补充国旗。
3. 系统需要补充赛程。
4. 天气数据和伤停数据缺失需要解决。

第一阶段目标是做一个可落地的“竞彩实战看板”，优先提升实战阅读效率和数据完整性提示，不引入复杂付费数据源，不改变现有模型主流程。

## 目标

第一阶段实现以下能力：

- 首页从普通表格升级为竞彩实战看板。
- 比赛名称显示中文队名和国旗。
- 新增赛程视图，支持按日期、轮次、场地查看比赛。
- 天气和伤停从简单缺失标签升级为明确的数据状态模块。
- 保持现有赔率导入、xG、Poisson、推荐标签和测试体系稳定。

## 非目标

第一阶段不做以下内容：

- 不接入付费伤停或赔率 API。
- 不承诺自动抓取所有国家队伤停。
- 不重写模型，不改变已有赔率方向和主客队计算逻辑。
- 不做复杂用户登录、权限、云端部署。

## UI 设计

采用“实战筛选型”页面结构。

### 顶部概览

顶部显示 4 个核心指标：

- 今日比赛数
- 强价值场次
- 高风险场次
- 数据缺失场次

这些指标来自最新一次 `model_runs` 对应的推荐结果和数据质量字段。

### 每日筛选

每日筛选区域从纯表格升级为更容易扫描的比赛列表或强化表格，显示字段：

- 比赛：国旗 + 中文主队 vs 中文客队 + 国旗
- 市场：主胜、平局、客胜、大 2.5 等中文市场名
- 推荐结论：强价值、轻仓观察、不下注、回避
- 模型优势
- 置信度
- 风险等级
- 主队 xG / 客队 xG
- 天气状态
- 伤停状态
- 简短说明

筛选控件保留：

- 推荐结论
- 风险等级
- 最低模型优势

新增可选筛选：

- 数据状态：完整、天气缺失、伤停缺失
- 比赛日期
- 轮次或小组

### 单场分析

单场分析区域保留现有指标，并补充：

- 比赛信息：国旗、中文队名、开球时间、场地、轮次/小组
- 概率分布：主胜、平局、客胜、大 2.5
- xG 指标：主队 xG、客队 xG、xG 差
- 天气模块：温度、湿度、风速、降水概率、更新时间、数据状态
- 伤停模块：球队、球员、影响位置、xG 修正、来源、置信度
- 风险说明：风险标签翻译为中文，并显示影响原因

## 国旗设计

第一阶段使用国旗 emoji，不引入图片资源。原因：

- 不需要额外静态文件和版权处理。
- Streamlit 原生支持显示。
- 对 48 支国家队维护成本低。

新增文件：

`data/manual/team_display.csv`

字段：

```text
team_name,display_name,country_code,flag_emoji
```

示例：

```text
荷兰,荷兰,NL,🇳🇱
日本,日本,JP,🇯🇵
乌兹别克,乌兹别克,UZ,🇺🇿
哥伦比亚,哥伦比亚,CO,🇨🇴
```

页面展示优先级：

1. `team_display.csv` 中的中文展示名和国旗。
2. 数据库 `matches.home_team` / `matches.away_team`。
3. `match_id` 作为兜底。

## 赛程设计

新增手工赛程文件：

`data/manual/schedule.csv`

字段：

```text
match_number,match_id,date,kickoff_time,round_name,group_name,venue_id,home_team,away_team
```

用途：

- 给页面提供赛程视图。
- 给天气抓取提供比赛时间和场地。
- 给后续小组战意、轮次战意提供基础结构。

数据合并规则：

- 赔率 CSV 仍是比赛和赔率的主输入。
- `schedule.csv` 用 `match_number` 或 `match_id` 补充日期、轮次、小组、场地。
- 如果 `schedule.csv` 没有某场比赛，页面仍展示赔率 CSV 中已有比赛，并标记“赛程信息缺失”。

## 天气数据设计

第一阶段继续使用已有 `app/pipelines/fetch_weather.py` 中的 Open-Meteo 方向。

需要补齐：

- `venues.csv` 中每个场地的经纬度、城市、时区。
- `schedule.csv` 中每场比赛的开球时间和 `venue_id`。
- 日报流程中按比赛场地和时间抓取天气并写入 `weather_snapshots`。

页面显示：

- 已获取：温度、湿度、风速、降水、天气代码、更新时间。
- 缺失：显示“天气数据缺失”，并说明缺少场地、时间或接口结果。

模型影响：

- 第一阶段只作为风险标签和数据质量影响。
- 不直接大幅调整 xG，避免天气数据早期不稳定导致模型误判。

## 伤停数据设计

第一阶段采用人工维护 + 置信度方案。

扩展 `data/manual/injuries.csv` 字段：

```text
match_id,team_name,player_name,position,status,xg_adjustment,note,source_url,source_confidence,updated_at
```

状态建议：

- out：缺阵
- doubtful：出战成疑
- limited：受限
- available：可出场

页面显示：

- 已确认伤停：显示球员、位置、状态、xG 修正、来源。
- 没有重大伤停：显示“暂无重大伤停记录”。
- 未维护：显示“伤停数据待确认”。

模型影响：

- 有 `xg_adjustment` 时进入现有 xG 修正逻辑。
- 没有明确数据时只增加风险标签，不自动臆测影响。

## 数据流

1. 运行 500 赔率抓取，生成 `jczq_500.csv`。
2. 运行日报管道。
3. 管道读取赔率、赛程、球队展示、场地、天气、伤停。
4. 管道写入 SQLite：
   - `matches`
   - `match_predictions`
   - `recommendations`
   - `weather_snapshots`
   - 伤停可继续从 CSV 在页面读取，或后续入库。
5. Dashboard 读取最新模型结果，合并比赛、国旗、赛程、天气、伤停状态。

## 错误处理

- 缺少赛程：比赛仍显示，赛程字段标记缺失。
- 缺少国旗：只显示中文队名。
- 缺少天气：显示天气缺失，并产生风险标签。
- 缺少伤停：显示伤停待确认，并产生风险标签。
- 外部天气接口失败：保留旧天气快照；没有旧快照时标记缺失。

## 测试计划

新增和调整测试：

- Dashboard 加载时能显示中文队名与国旗。
- 赛程 CSV 能合并到 `matches`。
- 缺少赛程时系统不崩溃。
- 天气缺失时显示中文风险状态。
- 伤停缺失时显示中文风险状态。
- 已有 `python -m pytest -q` 全量通过。
